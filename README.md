# Bombe

A historically accurate software recreation of the WWII Bombe machine used to crack Enigma-encrypted messages.

Implements correct Enigma mechanics (notch-based double-stepping, ring settings, plugboard) and the Bombe's physical circuit model with Welchman's Diagonal Board as constraint propagation.

---

## Requirements

- CMake 3.16+
- GCC 7+ or Clang 5+ (C++17)
- Internet access on first build (CMake FetchContent downloads Google Test)

```bash
sudo apt install cmake build-essential   # Ubuntu/Debian
```

## Scripts

All scripts run from the repo root and auto-build if the binary is missing.

| Script | Description |
|--------|-------------|
| `scripts/build.sh [Release\|Debug]` | CMake configure + build (default: Release) |
| `scripts/test.sh [--filter pattern]` | Run Google Test suite, optionally filtered |
| `scripts/generate.sh [options]` | Generate crib files with random Enigma settings |
| `scripts/crack.sh [crib] [options]` | Run the Bombe cracker |
| `scripts/clean.sh [--all]` | Remove `build/`; `--all` also removes `cribs/` |

### generate.sh

```bash
./scripts/generate.sh                          # one crib → cribs/bombe_pc.toml
./scripts/generate.sh --count 10              # 10 cribs → cribs/crib_<seed>.toml
./scripts/generate.sh --seed 42               # reproducible output
./scripts/generate.sh --count 5 --out-dir cribs/batch
```

### crack.sh

```bash
./scripts/crack.sh                             # default: cribs/bombe_pc.toml
./scripts/crack.sh cribs/crib_12345.toml       # single crib file
./scripts/crack.sh cribs/                      # all *.toml files in a directory
./scripts/crack.sh --ring-search               # search all 26² ring combinations
./scripts/crack.sh cribs/ --ring UPP           # override ring settings for every file
```

### test.sh

```bash
./scripts/test.sh                              # run all 14 tests
./scripts/test.sh --filter "EnigmaTest.*"      # run only Enigma tests
./scripts/test.sh --filter "BombeTest.RecoverKnownSettings"
```

### Ring settings flags

| Flag | Behaviour |
|------|-----------|
| *(none)* | Use ring settings embedded in the crib file; fall back to `settings.toml` |
| `--ring ABC` | Override with known ring settings (e.g. `--ring UPP`) |
| `--ring-search` | Search all 26² left/middle rotor ring combinations |

## Configuration

### `config/machine.toml` — Rotor and reflector definitions

```toml
[[rotors]]
name   = "I"
wiring = "EKMFLGDQVZNTOWYHXUSPAIBRCJ"
notch  = "Q"   # window position that triggers the left neighbour to step

[[reflectors]]
name   = "UKW-B"
wiring = "YRUHQSLDPXNGOKMIEBFZCWVJAT"
```

Includes the five historical Wehrmacht/Luftwaffe Enigma I rotors (I–V) and the UKW-B reflector.

### `config/settings.toml` — Bombe search options

```toml
[bombe]
max_scramblers       = 0      # 0 = unlimited; 36 = physical Bombe limit
search_ring_settings = false  # true searches 26² ring combinations (slow)
fixed_ring_settings  = [0, 0, 0]

[enigma]
reflector = "UKW-B"
# rotors = ["I", "II", "III"]   # uncomment to restrict rotor search
```

### Crib file (`cribs/bombe_pc.toml`)

Generated by `generate_crib`. The file is self-contained: it embeds the ring settings and reflector used during encryption so the Bombe can use them without manual configuration.

```toml
# Bombe crib — generated with seed 42
# Reproduce: ./build/cli/generate_crib --seed 42

[settings]
ring_settings = [20, 15, 15]
reflector     = "UKW-B"

[[cribs]]
plain  = "WESTDESTROYER"
cipher = "CUTRIVYAYRIIW"
```

Each crib pair is an independent message encrypted from the same Enigma starting position. The legacy CSV format (`PLAIN,CIPHER` per line) is also accepted for backward compatibility.

## How it works

### Enigma simulation

- **Rotor stepping**: notch-based with the double-stepping anomaly (middle rotor steps twice when at its notch position)
- **Ring settings (Ringstellung)**: shift rotor internal wiring relative to the alphabet ring; independent of notch position
- **Plugboard**: applied before and after the rotor path
- **Signal path**: right → left through rotors → reflector → left → right (involution)

### Bombe algorithm

1. **Build menu** — each plaintext/ciphertext character pair creates an edge in the menu graph
2. **Find cycles** — DFS cycle detection; longer cycles give stronger constraints
3. **Select test letter** — the letter with the highest degree in the menu graph
4. **For each rotor combination × starting position**:
   - Try all 26 possible plugboard connections for the test letter
   - Propagate constraints through the scrambler circuit (BFS)
   - **Diagonal Board**: enforces plugboard symmetry `plug[a]=b ↔ plug[b]=a` at each step
   - Contradiction → eliminate; no contradiction → **stop**
5. **Verify stop** against all crib texts to filter false positives

### Search space

| Dimension | Size | Notes |
|---|---|---|
| Rotor permutations | 60 | P(5,3) |
| Starting positions | 17,576 | 26³ |
| Ring settings | 676 (optional) | 26² — rightmost fixed at 0 in search mode |
| Plugboard assumptions | 26 | per position |

## Note on deprecated/

The original implementation (`deprecated/`) used simple linear stepping (pure odometer, no notch) instead of the historically correct double-stepping mechanism. Cribs generated with that code (`deprecated/bombe_pc.txt`) are not compatible with this implementation. Use `generate_crib` to produce correct test data.
